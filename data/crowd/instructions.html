In this task, imagine a user who wants to find information online and unintentionally asks a search system (like Google or Siri) an ambiguous search query. To better understand their (unspecified) search intention, the system asks a clarifying question.

For example, a user may be looking for an anti-spyware program called Defender and may ask "Tell me about defender". However, the query is ambiguous: the word "defender" may refer to many things: a TV series ‘The Defender’, a vehicle named ‘Defender’ and so on.

Perhaps the system may guess that the user is looking for the TV series, and asks: "Are you interested in a television series?"

Now, the user may answer the question in many ways. It can be as simple as “No.” or a longer “No, I am looking for the computer program Defender.” depending on the user’s personal preferences.

The goal of this task is to evaluate the relevance and naturalness of answers to these clarifying questions. You will be given the (possibly ambiguous) search query, the user’s search intent (which is invisible to the search system) and the clarifying question. For each question, you will receive a few user responses which may or may not answer the question.

Your goal is to assign Naturalness and Relevance scores (in the range 1-5) for each answer.

Relevance

The Relevance of an answer measures whether the answer is aligned with the user’s search intent. In other words, it helps the search system better understand the user’s unspecified intention. An answer that is perfectly aligned with the intent should receive a relevance score of 5. An answer that contradicts the search intent or discusses unrelated topics should receive a relevance score 1.

In the above example, both “No.” and “No, I am looking for the computer program Defender.” perfectly capture the user’s unspecified intention and should receive a relevance score of 5. Answers such as “Yes.”, “I am interested in the television series.” or “I didnt know defender was a TV series” should receive a relevance score of 1 because they have no connection to the user’s original search intent (i.e. the anti-spyware program).

Naturalness

The Naturalness of an answer rates whether the answer is fluent, grammatical, and human-like. A perfectly natural answer could have been written by a reasonably fluent English speaker, and does not provide information beyond what the question is asking for. Such answers would receive a naturalness score of 5. An unnatural answer might have logical errors in them, or perhaps be impossible to understand. Such answers would receive a naturalnes score of 1.

In the above example, both “No.” and “No, I am looking for the computer program Defender.” would receive a naturalness score of 5. Answers such as “Yes, I am looking for the computer program.” or “No, I am looking for the television series” would receive naturalness scores of 1. Recall that the user wants to find a computer program and the search system asked if they were looking for a television series, and no reasonable user would have written the above answers.

Note that while the answer “Yes, I am looking for the computer program.” is not natural, it can help the search system understand the user’s intent and would receive a high relevance score.

In terms of grammatical, please ignore mistakes caused by cases and "'", for example, do not take points off for using "im" for "I'm".

Differences between Relevance and Naturalness

Note that relevance and naturalness are not dependent.

If an answer is aligned with the user's search intent, but does not answer what the question is asking for, then it is relevant but not natural.

If an answer is a possible answer for the clarifying question, but contradicts the user's search intent, then it is natural but irrelevant. For example, for a yes-no question, both yes or no could be natural, but at least one of them must be irrelevant.

Examples

Here are some examples with explanations:

Example 1: The clarifying question matches the search intent

    Search intent: "Anti-spyware program Defender"

    User Query: "Tell me about defender"

    Clarifying question: “Are you interested in the software Defender"

    ✔ Good answers:

    • “Yes”
    • “Yes, that is what I need”
    • “The anti-spyware program”

    ✘ Bad answers:

    • “No” (Explanation: This answer contradicts with the user's search intent, so it is bad because it has low relevance. However, it is actually natural because it can be otherwise true.)
    • “I don’t know” (Explanation: This is neither relevant nor natural, because the user should know and answer “yes”)

Example 2: The clarifying question is relevant but does not match the search intent

    Search intent: "anti-spyware program Defender"

    User Query: "Tell me about defender"

    Clarifying question: "are you interested in the television series Defender"

    ✔ Good answers:

    • “No”
    • “No that is not what I am looking for”
    • “No i want troubleshooting information for windows defender the software”

    ✘ Bad answers:

    • “Yes, that is what I want” (Explanation: No, it is not what the user wants, it contradicts with the search intent. It is natural but not relevant)
    • “I don’t know” (Explanation: Because the user should know it. This is neither natural nor relevant)

Example 3: The clarifying question is not a well-formed clarifying question

    Search Intent: "Find vendors that sell computer memory upgrades."

    User Query: "memory"

    Clarifying question: “who was the first to study the brain and memory”

    Note that the clarifying question is irrelevant to the user’s query.

    ✔ Good answers:

    • “It is not related to my search”
    • "I don’t know”
    • “Don’t ask me”

    ✘ Bad answers:

    • “Hermann Ebbinghaus” (Explanation: literally answering the question asked by the system. This is bad because it is unnatural to do so when the user is the one searching for information.)
    • “Yes”/“No” (Explanation: yes or no cannot be used to answer these questions)
